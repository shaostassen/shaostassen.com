# S0.3 — Design system foundation

Status: done · Started: 2026-07-08 · Shipped: 2026-07-08

## Ideate

Theming approaches considered: (a) Tailwind `dark:` variants with a class
toggle, (b) duplicated variable blocks under a media query + data-attribute
override, (c) `light-dark()` CSS function with `color-scheme`. Picked (c):
tokens switch themselves, components never write `dark:` variants at all,
and the S1.1 toggle reduces to setting `data-theme` on `<html>` — no
storage, no FOUC script. Browser support is universal since 2024.

Fonts: Space Grotesk (display) + Inter (body) + JetBrains Mono (technical
accents) over keeping Geist (too "Vercel default") or IBM Plex (plainer
headings). Accent: restrained cyan (#0e7490 light / #22d3ee dark) — a nod
to a scope trace without phosphor-green cosplay.

## Decisions

- Semantic tokens (`background/surface/border/foreground/muted/accent`) in
  `src/styles/globals.css`, mapped to utilities via `@theme inline`.
- Prose styles hand-rolled from tokens rather than @tailwindcss/typography —
  no plugin dep, no `prose-invert`, automatically theme-correct.
- `/styleguide` gated by `notFound()` when `NODE_ENV === "production"`:
  the exported `styleguide.html` contains only 404 content (verified, and
  pinned by a smoke test).
- Type scale: Tailwind defaults + `text-display` (clamp 2.5–3.25rem) and
  `text-title` (1.5rem).
- Global `prefers-reduced-motion` kill-switch in globals.css.
