# S1.1 — App shell

Status: done · Started: 2026-07-08 · Shipped: 2026-07-08

## Ideate

Theme persistence: the plan's acceptance said "no browser-storage
dependency", but a toggle that resets on every full page load isn't a
toggle — static export means each navigation can be a fresh document.
Deviation, reasoned: localStorage with system preference as the default
(no stored value → system decides), applied by a 1-line inline script
before first paint (no flash). This is the industry-standard pattern; the
plan's parenthetical read like a leaked constraint from a different
context.

Nav links: ship only destinations that exist (wordmark → /, GitHub,
toggle). Page links join the `links` array as their routes land — the nav
never points at a 404.

## Decisions

- ThemeToggle has no theme state: which icon shows is pure CSS
  (`light-dark()` display swap via `.show-in-light/.show-in-dark`), so
  there is no hydration mismatch and no mounted-flag dance. JS only
  handles the click.
- Hide-on-scroll via rAF-throttled scroll listener; `onFocusCapture` on
  the header reveals it whenever keyboard focus enters, so tabbing never
  lands in an off-screen landmark.
- Skip link is the first tab stop, targets `main#content` (now owned by
  the layout; pages render content only, no `<main>` of their own).
- Footer socials: GitHub only until Shao confirms LinkedIn/email
  (docs/drafts/REVIEW.md); "built with + source" per portfolio-as-project.

## Gotchas

- Tailwind 4's `-translate-y-full` sets the CSS `translate` property, not
  `transform` — verify with `computedStyle.translate`.
- The preview browser's eval context reported contradictory scroll state;
  verified with a real Playwright run instead (hide on down ✅, reveal on
  up ✅, reveal on focus ✅).
