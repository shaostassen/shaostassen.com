# S2.1 — Typed content layer + one seeded project

Status: done · Started: 2026-07-08 · Shipped: 2026-07-08

## Ideate

MDX pipeline options: (a) @next/mdx with pages authored in app/, (b)
next-mdx-remote/rsc over a content directory, (c) hand-rolled remark
pipeline. Picked (b): content lives in `src/content/projects/` per the
plan's architecture, compiles in server components at build time, zero
client JS, and frontmatter parses into a plain object that Zod can
validate — which is the acceptance criterion with teeth.

Seed project: **Fast Robots** — the only fully confirmed draft after
Shao's review pass (timeframe, report-site URL, stunts, approach).

## Schema decisions

- `track: "school" | "individual"` added per Shao's 2026-07-08 direction
  (site splits into School work — HS + college — and Individual work).
  S3.1 groups by it; recorded as a plan amendment in STORIES.md.
- `status: "complete" | "in-progress"` (SpeechLens will need it; the
  detail page renders an "in progress" marker).
- Experience/skills/education schemas defined (acceptance) — data modules
  arrive with S4.x once facts are confirmed.
- Loader validates at build: bad frontmatter → ZodError → build fails.
  Proven live: `category: not-a-category` produced the exact enum error
  and a failed build. Slug/filename mismatch also throws.
