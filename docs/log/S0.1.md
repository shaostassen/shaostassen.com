# S0.1 — Repo + scaffold + first live deploy

Status: done · Started: 2026-07-07 · Shipped: 2026-07-08

## Ideate

Three ways to scaffold: (a) create-next-app with flags, (b) hand-rolled minimal
Next setup, (c) clone a starter template. Picked (a): it's the canonical path,
stays closest to upstream defaults (less to explain in the colophon), and the
plan's stack matches its output exactly. Pinned to create-next-app@15 to honor
the plan's Next.js 15 choice (16 is out).

## Plan / Architect

- Scaffold into the existing `~/Documents/shaostassen.com` (docs/ already held
  the plan + drafts; create-next-app allowlists `docs`).
- `git init` before scaffolding so create-next-app skips its auto-commit and
  the history stays hand-authored.
- `output: 'export'` + `images.unoptimized` from day one (host-portability rule,
  PLAN §3). Placeholder landing kept deliberately minimal — real design system
  lands in S0.3.

## Build notes / gotchas

- corepack is broken on Node 25 (`ERR_VM_DYNAMIC_IMPORT_CALLBACK_MISSING`);
  installed pnpm 11.10.0 via `npm i -g --prefix ~/.local` instead.
- pnpm 11 blocks dependency build scripts by default; approved `sharp` and
  `unrs-resolver` via `allowBuilds` in pnpm-workspace.yaml (note: the newer
  `allowBuilds` schema, not `onlyBuiltDependencies`).
- Machine keychain held a token for the `ses439` GitHub account, so the repo
  was first created there; decision: portfolio lives on `shaostassen`.
  Initiated an API repo transfer ses439 → shaostassen (accepted 2026-07-08).
- Vercel account needed two browser-side grants before `vercel git connect`
  succeeded: a GitHub login connection, then the Vercel GitHub App install.
