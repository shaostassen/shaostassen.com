# S0.2 — Claude Code harness

Status: done · Started: 2026-07-08 · Shipped: 2026-07-08

## Ideate

Options for the QA tooling: (a) Lighthouse CI (`@lhci/cli`) with
`staticDistDir`, (b) unlighthouse, (c) raw lighthouse CLI + jq. Picked (a):
it serves `out/` itself (no extra server), has first-class assertion budgets,
and its config reads as standard practice. Playwright tests run against the
built static export via `serve` rather than the dev server — what passes is
what ships.

## Decisions

- `CLAUDE.md` is public repo content: project rules only. Machine quirks
  (PATH, ssh-add, account gotchas) live in gitignored `CLAUDE.local.md`.
- Six skills under `.claude/skills/`, committed — they encode the design
  ground rules now and S0.3 fills in concrete token values.
- Git flow per decision: direct to main, terse conventional one-liners, no
  attribution trailers.
- Prettier with default config; `docs/` excluded to avoid churning the
  human-authored plan.
- STORIES.md keeps only ID/epic/title/status/result — acceptance criteria
  stay in PLAN §11 to avoid drift between two copies.

## Gotchas

- Playwright `iPhone 13` device preset implies WebKit, which isn't
  installed; mobile project switched to `Pixel 7` (Chromium) — the smoke
  test needs a small viewport, not Safari.
