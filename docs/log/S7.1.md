# S7.1 — Accessibility & responsive hardening

Status: done · Started: 2026-07-12 · Shipped: 2026-07-12

## What was added

- **axe-core suite** (tests/a11y.spec.ts): WCAG 2.1 A/AA scan of all nine
  routes (incl. 404) × light/dark × desktop/mobile viewports, plus the
  open mobile menu. 40 checks, all green.
- **Escape-to-close** on the mobile disclosure menu, with focus returned
  to the menu button (real gap in the disclosure pattern).
- **Reduced-motion regression test** — and a real bug it caught
  immediately: the global kill-switch zeroed animation durations but not
  delays, so reduced-motion users saw staggered invisible content for up
  to 240 ms. animation/transition delays now zeroed too.

## Notes

- Axe initially flagged color-contrast on the landing hero: it measures
  blended colors mid-entrance-fade. Tests now wait for
  `document.getAnimations()` to finish — steady-state is what matters,
  and the animation is 500 ms, opacity-only, and reduced-motion-gated.
- Keyboard coverage now spans: skip link first tab stop, focus-revealed
  hide-on-scroll nav, menu button semantics (aria-expanded + Escape +
  focus return), aria-pressed filter chips. All focus states use the
  visible accent outline.
- Responsive: 375/768/1440 verified across pages (768 coursework/contact
  swept this pass).
