## Problem

[CONFIRM: the motivating use — what does SpeechLens analyze speech *for*? Transcription review, filler-word/pacing feedback, meeting notes?] Whatever the analysis, the design constraint was non-negotiable: audio never leaves the machine. No cloud ASR APIs — full quality, locally.

## Constraints

- Two very different deployment targets from one codebase: an RTX 5090 workstation with VRAM to burn, and a Jetson Orin Nano where every gigabyte and watt counts.
- Real audio is messy — silence, crosstalk, and noise mean you can't just feed raw recordings to an ASR model and trust the output.
- Usable by a human: a web UI, not a script with flags.

## Approach

A staged pipeline: Silero VAD segments speech from silence and noise first, so the ASR model only sees speech and timestamps stay honest; faster-whisper (CTranslate2-backed Whisper) does transcription with a model size selected per target; a FastAPI service exposes the pipeline; a web UI sits on top. [CONFIRM: streaming vs. batch processing, which Whisper model sizes per device, and what the analysis layer computes.]

## Why it's technically hard

The single-codebase, two-targets requirement is the real problem. What's trivially fast on a 5090 has to be *engineered* to fit the Orin Nano — model size, quantization, and memory pressure all have to give, without forking the code. VAD-before-ASR sounds like a detail but drives output quality: hallucinated transcription in silence is Whisper's best-known failure mode, and segmentation is what prevents it.
